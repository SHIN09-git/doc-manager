# P2 第四轮商业化施工文档

本文件承接 `P2_ROUND3_COMMERCIALIZATION_BUILD_PLAN.md`。前三轮已经补齐邮件 webhook、支付签名与价格映射、管理后台二级页、备份校验与失败告警、组织治理和 PostgreSQL 拆分方案。第四轮目标是继续向“可收费灰度”推进。

## 施工顺序

```text
1. 支付 checkout 与账单页入口
2. 邮件投递回调与运营筛选
3. 管理后台独立页面雏形
4. PostgreSQL repository 试点
5. 备份加密与对象存储方案
6. 文档、测试与 review
```

## P2R4-1：支付 checkout 与账单页入口

要做：

- 新增云端账单页入口，展示当前套餐、用量、升级按钮和账单事件。
- 后端新增创建 checkout session 的适配接口，占位支持真实支付渠道。
- 前端升级按钮走后端接口，不直接拼接支付链接。

验收：

- 免费用户能看到升级入口。
- 后端返回 checkout URL 或明确的未配置提示。
- 支付配置缺失时不暴露内部错误。

## P2R4-2：邮件投递回调与运营筛选

要做：

- 增加邮件服务商回调接口草案，用于记录 delivered/bounced/failed。
- 管理后台增加邮件投递列表筛选。
- 邮件失败可复制详情。

验收：

- 邮件状态可以从 `sent` 更新为服务商回调状态。
- 管理员能按模板、状态查看邮件投递。

## P2R4-3：管理后台独立页面雏形

要做：

- 在不破坏现有云端面板的前提下，增加独立管理页面入口或 hash 路由。
- 将组织概览、用量、审计、反馈、错误拆成更清晰的二级视图。
- 保持 owner/admin 权限判断。

验收：

- 管理后台不再完全依赖右侧面板空间。
- 原弹窗入口仍可用或能跳转到独立页面。

## P2R4-4：PostgreSQL repository 试点

要做：

- 按 `POSTGRES_REPOSITORY_REFACTOR_PLAN.md` 先实现迁移版本表和一个只读 repository。
- 优先试点 `ai_usage` 或 `audit_logs`，降低业务写入风险。
- 增加轻量测试。

验收：

- 不影响 JSON Store。
- PostgreSQL Store 启动能记录迁移版本。
- 试点 repository 有测试覆盖。

## P2R4-5：备份加密与对象存储方案

要做：

- 出具对象存储和加密备份配置方案。
- 备份脚本预留加密/上传接口，未配置时保持本地备份。
- 文档说明恢复前的密钥和权限检查。

验收：

- 文档清楚说明备份文件不应明文长期落盘。
- 脚本保持向后兼容。

## 非目标

本轮不做：

- 完整发票系统。
- 企业级工单系统。
- PostgreSQL 全量 repository 改造。
- 多租户高并发压测平台。
