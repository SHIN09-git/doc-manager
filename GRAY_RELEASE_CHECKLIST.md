# 灰度试用清单

本清单用于 P1 第二轮之后的小范围商业化试用。

## 上线前

- [ ] 已配置生产强密钥和明确 `CORS_ORIGIN`。
- [ ] 已运行 `npm run build`、`npm run check`、`npm test`、`npm run test:e2e`。
- [ ] 已验证 `/api/health` 和 `/api/ready`。
- [ ] 已确认数据库备份策略。
- [ ] 已准备隐私政策草案和用户协议草案。
- [ ] 已准备故障回退说明。

## 内测用户

- [ ] 邀请 3 到 5 个真实用户。
- [ ] 每个用户完成邮箱验证。
- [ ] 每个组织至少一名 owner。
- [ ] 明确告知不要上传涉密或未经授权材料。

## 试用任务

- [ ] 导入文档并保存到云端。
- [ ] 生成或同步一个执笔人。
- [ ] 使用云端 AI 代理生成一篇文档。
- [ ] 触发并处理一次版本冲突。
- [ ] 邀请和移除一个成员。
- [ ] 提交一条内测反馈。

## 每日检查

- [ ] 查看 AI 用量和任务类型分布。
- [ ] 查看最近错误。
- [ ] 查看审计日志。
- [ ] 汇总用户反馈到 TODO 或 issue。

## 回退

- [ ] 云端故障时通知用户切回本地 AI 接口。
- [ ] 暂停团队邀请和云端 API Key 更新。
- [ ] 从最近备份恢复数据库。
