# 摹文拟笔工作台商业化准备清单

本文档用于记录将「摹文拟笔工作台」从本地/开源工具推进到可在网站上商业化运营时需要补齐的工作。

建议商业化路线：

```text
开源基础版
+
付费云端增强版
+
组织/学校私有化部署
```

核心定位不应只是“AI 生成文档”，而应突出：

> 把一个单位长期沉淀的写作规则，变成可管理、可复用、可审计的执笔人体系。

## P0：上线前必须补齐

### 1. 账号系统

- 用户注册、登录、退出
- 找回密码
- OAuth 登录，可选
- 组织空间
- 成员管理
- 用户身份与订阅状态绑定
- 账号注销与数据删除流程

### 2. 后端服务

当前项目偏前端本地工作台。商业化版本需要后端承接：

- 用户账户
- 文档库
- 执笔人库
- 文件上传
- AI 调用代理
- 付费状态
- 配额统计
- 操作日志
- 后台管理接口

### 3. 数据存储策略

需要明确三种使用模式：

- 本地优先：文档只存在浏览器或本机。
- 云同步：文档加密后同步到云端。
- 私有部署：机构自行部署数据库、对象存储和模型接口。

文档类产品最重要的是用户信任，需要清楚说明：

- 文档是否上传云端
- 上传后保存多久
- 是否会进入模型训练
- 删除后是否真正删除
- 是否支持本地模型或内网模型

### 4. AI 接口商业化设计

建议保留两种模式：

- 用户自带 API Key：降低平台成本和合规压力。
- 平台托管 API：平台统一计费、限额、风控和模型选择。

需要实现：

- API Key 加密存储
- 模型供应商配置
- 每用户 / 每组织调用额度
- Token 使用统计
- 失败重试上限
- 任务取消
- 模型异常的友好错误提示
- 大文件与高成本任务拦截

参考：

- [OpenAI API data controls](https://platform.openai.com/docs/models/how-we-use-your-data)
- [OpenAI business data privacy](https://openai.com/business-data)

### 5. 计费系统

海外优先可考虑 Stripe Checkout。

需要实现：

- 套餐订阅
- 支付成功回调
- 订阅状态同步
- 取消订阅
- 续费失败处理
- 额度重置
- 发票或账单记录

国内商业化还需要考虑：

- 微信支付
- 支付宝
- 发票
- 对公转账
- 私有化合同付款

参考：

- [Stripe Checkout 文档](https://docs.stripe.com/payments/checkout)

### 6. 安全基线

至少需要：

- HTTPS
- 密码哈希
- API Key 加密存储
- 文档访问鉴权
- 组织数据隔离
- 上传文件大小限制
- 上传文件类型限制
- 防 XSS
- 防 CSRF
- 请求限流
- 审计日志
- 数据备份
- 数据恢复
- 删除数据机制
- 管理后台权限隔离

参考：

- [OWASP ASVS](https://github.com/OWASP/ASVS)

## P1：产品商业化能力

### 1. 定价与套餐

建议先做三档：

| 套餐 | 适合用户 | 核心能力 |
| --- | --- | --- |
| Free | 个人试用、开源用户 | 本地使用、少量执笔人、用户自带 API Key |
| Pro | 高频个人或小团队 | 云同步、更多执笔人、更多 AI 调用、PPT/Word 高级导出 |
| Team | 学校、部门、机构 | 团队执笔人库、权限、审计、共享模板、统一计费 |

### 2. 官网与转化页面

需要补齐：

- 首页
- 产品介绍页
- 价格页
- 文档页
- 安全说明页
- 开源说明页
- 登录 / 注册入口
- 联系销售 / 私有化部署入口

首页重点内容：

- 产品一句话定位
- 典型使用场景
- 执笔人能力演示
- 文档安全说明
- 价格
- FAQ
- 开源地址
- 开始使用按钮

### 3. 新用户引导

需要降低第一次使用门槛：

- 首次打开引导
- 示例文档
- 示例执笔人
- “上传 5 篇文档生成执笔人”的流程引导
- 生成前后对比演示
- 空状态引导
- 错误恢复提示

### 4. 团队协作

执笔人是核心差异化能力，商业化后应重点做团队化：

- 团队执笔人库
- 执笔人导入 / 导出
- 执笔人版本回退
- 谁创建、谁修改、谁启用
- 组织内共享
- 管理员审核后发布
- 团队模板库
- 团队文风规范库

### 5. 成本控制

AI 产品必须有成本墙：

- 用户额度
- 组织额度
- 月度额度重置
- 大文件限制
- 生成任务排队
- Token 统计
- 模型分级
- 失败重试上限
- 高成本任务二次确认
- 月账单统计

## P2：合规与信任

### 1. 必备法律与产品文档

需要准备：

- 用户协议
- 隐私政策
- Cookie 政策
- 退款政策
- 数据删除说明
- 开源许可证说明
- AI 生成内容免责声明
- 安全白皮书或数据安全说明
- 商业授权说明

### 2. 面向学校、政府、企事业单位的信任说明

需要重点说明：

- 用户文档是否上传云端
- 是否用于模型训练
- API Key 是否加密
- 删除后是否真的删除
- 是否支持私有化部署
- 是否支持内网模型
- 是否支持统一审计
- 是否支持数据导出

### 3. 国内公开网站合规事项

如面向中国大陆公开运营，需要评估：

- ICP 备案
- 公安备案
- 个人信息保护法相关要求
- 数据出境问题
- 生成式 AI 服务相关要求
- 支付与发票合规

说明：此部分需要正式上线前咨询律师或合规顾问。

### 4. 可访问性

建议按 WCAG 2.2 做基本检查：

- 键盘可操作
- 焦点状态清楚
- 文本对比度合格
- 表单有明确标签
- 弹窗焦点管理
- 错误提示可被读屏识别

参考：

- [W3C WCAG 2.2](https://www.w3.org/WAI/standards-guidelines/wcag/new-in-22/)

### 4. 客服与反馈

## 推荐施工顺序

### 阶段 1：商业模式确认

- 明确开源基础版边界
- 明确 Pro 版付费能力
- 明确 Team / 私有化能力
- 写定价草案
- 写目标用户画像

### 阶段 2：技术架构升级

- 拆分前端、后端 API、数据库、文件存储
- 设计用户、组织、文档、执笔人、订阅、额度的数据表
- 接入登录系统
- 接入 AI 调用代理
- 加入基础审计日志

### 阶段 3：云端执笔人体系

- 云端执笔人库
- 团队共享
- 版本管理
- 导入 / 导出
- 启用 / 停用
- 权限管理

### 阶段 4：支付闭环

- 订阅套餐
- 支付页面
- 支付回调
- 额度开通
- 续费失败处理
- 管理后台查看订阅

### 阶段 5：官网与转化

- 首页
- 价格页
- 安全说明页
- 文档页
- 开源页
- 登录注册页
- 私有化联系入口

### 阶段 6：内测与运营

- 邀请内测用户
- 收集反馈
- 修复高频问题
- 补帮助文档
- 完成正式发布清单

## 最小商业化 MVP 建议

如果要尽快上线第一个收费版本，建议最小范围是：

- 登录注册
- 用户自带 API Key
- 云端保存执笔人
- 云端保存少量文档
- Pro 订阅
- 每月额度
- 执笔人导入 / 导出
- 基础安全与隐私说明
- 官网首页 + 价格页

先不要急着做：

- 复杂权限矩阵
- 多模型市场
- 复杂模板市场
- 企业级审计全量功能
- 超完整协同编辑

## 风险提醒

- 文档和公文材料通常敏感，隐私与数据安全是第一销售门槛。
- AI 成本如果不做额度和模型分级，很容易吞掉收入。
- 商业化前需要明确开源版和付费版边界，避免用户觉得核心能力被突然收走。
- 机构客户更关心稳定、安全、私有化和售后，不只关心生成效果。
