# 摹文拟笔工作台

本地优先的中文正式文本管理、AI 起草、执笔人构建与 PPTX 生成工作台。

它面向经常处理通知、总结、方案、纪要、请示、函件、讲话稿、汇报材料等正式文本的人。项目默认不绑定某个行业，可以用于学校、机关、企业、社群和个人知识工作场景。

```text
导入文档 -> 构建执笔人 -> 用 @ 调用 -> 稳定生成同类文本
```

## 功能概览

| 模块 | 能力 |
| --- | --- |
| 文档管理 | 本地创建、导入、编辑、手动排序、Word 导出、备份文档和垃圾箱恢复，支持标签和真实文件夹关联 |
| AI 起草 | 在大提示词窗口中输入写作任务，可用 `@执笔人` 控制结构和文风，也可把左侧文档卡片直接拖入作为素材 |
| 便捷编辑 | 支持查找替换、右键复制/删除、按执笔人快速改写段落、全文格式整理，编辑区宽度可拖拽调整 |
| 执笔人工作台 | 从多篇同类样本文档中提炼可复用、可编辑、可测试、可迭代、可导入导出的写作规则 |
| PPT 生成 | 内置归藏杂志 / 瑞士变体与公文汇报风格，支持手动或自动页数、风格、放大预览、结构自检，并生成可编辑的原生 `.pptx` |
| 文件导入 | 支持文本、Word、PPTX 拖拽导入，并提取 `.docx` / `.pptx` 中的表格 |
| 响应式布局 | 桌面三栏、窄屏右侧功能抽屉、手机“文档 / 编辑 / 功能”单栏切换 |
| 键盘辅助 | 文档列表支持方向键与回车选择，编辑菜单支持方向键与 Esc，PPT 预览弹窗支持焦点回位 |
| AI 接口 | 支持 OpenAI 兼容 Chat Completions 接口，由用户自行配置 API Key |

## 执笔人是什么

“执笔人”是项目里的写作能力单元。它不是简单保存范文，而是把多篇同类文本转化为可复用规则，用于后续自动生成同类文档。

执笔人构建链路：

1. 单篇文档解析：提取结构、文风、句式、格式、变量槽位和禁止复用内容。
2. 多篇文档聚合：比较共同点、差异点和冲突点。
3. 执笔人草案生成：把稳定规则整理成可复用写作能力。
4. 人工校准：允许用户修改、删除、增强规则。
5. 版本管理：每次优化生成新版本，可查看、回退和比较。
6. 生成测试：用执笔人生成测试文档，并检查规则命中。
7. 持续优化：继续添加样本或反馈生成结果，让执笔人迭代。

系统会同时生成两类产物：给人看的简短说明，以及给程序调用的执行卡。执行卡包含触发说明、输入契约、结构模板、必守规则、可选表达、执行流程、禁忌和自检清单，后续文档生成只读取必要规则，避免把样本原文整段塞进提示词。

核心原则：

> 单篇文档只能产生候选规则，多篇文档共同验证后才能成为强规则。

系统会尽量避免把具体人名、时间、地点、活动名称、临时安排、一次性政策和隐私信息当成通用写作规则。

日常编辑时也可以直接使用执笔人：在正文中选中一段，右键选择执笔人，再用“保留原意 / 改文风 / 扩写 / 缩写”四个预设快速改写。这个入口适合“把当前这段改成 @会议纪要 风格”一类轻量任务。

执笔人可以导出为 `.skill.json` 独立包，包含规则 JSON、说明 Markdown、质量报告、版本摘要和示范文件摘要，不导出原始示范全文。导入同事分享的 `.skill.json` 后，会自动处理 `@调用名` 冲突。

## 快速开始

### 方式一：直接打开

构建后的项目可以直接双击打开：

```text
index.html
```

页面加载的是已经打包好的 `build/bundle.js`，不需要后端服务。

### 方式二：本地服务器

如果浏览器策略限制 `file://`，可以使用本地静态服务器：

```bash
npm install
npm run build
npx serve .
```

也可以使用脚本启动：

```bat
start.bat
```

```bash
./start.sh
```

## 基本使用

1. 点击右上角“接口”，配置 OpenAI 兼容接口。
2. 点击“导入”或把 `.txt`、`.md`、`.csv`、`.docx`、`.pptx` 拖入文档区。
3. 在中间编辑当前文档，点击“保存当前文档”保存到本机浏览器存储。
4. 删除文档会先进入右上角“垃圾箱”，可在独立窗口中单个或批量恢复 / 清除。
5. 点击文档只会选中高亮，不改变顺序；需要调整顺序时可拖拽文档卡片，或在卡片二级菜单中置顶 / 置底。
6. 在“执笔人”栏目添加多篇同类示范文本，构建写作规则。
7. 在“生成”栏目输入任务，使用 `@执笔人名称` 调用对应能力，也可以把左侧文档直接拖入提示词窗口。
8. 选中正文段落后右键，可用当前执笔人做快速改写、扩写或缩写。
9. 在“执笔人”栏目右上角可导入 / 导出 `.skill.json`，用于同事共享或仓库版本管理。
10. 在“PPT”栏目输入材料或拖入 PPTX / Word / 文本，生成并下载原生 `.pptx`。

## AI 接口

项目不内置 API Key。点击右上角“接口”按钮后配置：

| 配置 | 说明 |
| --- | --- |
| Base URL | OpenAI 兼容接口基础地址，例如 `https://api.openai.com/v1` |
| Endpoint Path | 通常为 `/chat/completions` |
| Model | 使用的模型名称 |
| API Key | 你的接口密钥，仅保存在当前浏览器环境 |
| System Prompt | 默认系统提示词，可按组织习惯调整 |

AI 调用链路带有重试、友好错误提示、进度提示、长任务取消和 JSON 容错修复。

## 导入与导出

当前支持导入：

- `.txt`
- `.md`
- `.text`
- `.csv`
- `.docx`
- `.pptx`

说明：

- `.txt` / `.csv` 会尝试识别 UTF-8、GB18030 / GBK、UTF-16 等常见编码。
- `.docx` 会读取普通段落和表格，表格会转换为 Markdown 表格。
- `.pptx` 会读取幻灯片文本、备注和表格，表格同样会转换为 Markdown 表格。
- 旧版 `.doc` 和 `.ppt` 暂不支持，可先用 Word、PowerPoint 或 WPS 另存为 `.docx` / `.pptx`。

导出能力：

- 当前文档默认导出为 `.docx`，导出时会去掉正文开头与文档标题重复的 Markdown 大标题
- 工作台数据导出为备份 `.json`
- PPT 生成模块导出为可编辑的原生 `.pptx`

## 开发命令

```bash
npm install
npm run build
npm run check
npm test
npm run test:e2e
```

| 命令 | 作用 |
| --- | --- |
| `npm run build` | 使用 esbuild 从 `src/main.js` 打包到 `build/bundle.js` |
| `npm run check` | 检查入口脚本和打包产物语法 |
| `npm test` | 运行 Node 单元测试 |
| `npm run test:e2e` | 构建后启动托管静态服务，并运行 Playwright 浏览器测试 |

修改 `src/` 后需要重新运行 `npm run build`，因为浏览器实际加载的是 `build/bundle.js`。
E2E 入口由 `e2e/run-e2e.mjs` 管理静态服务生命周期，测试结束后会主动关闭服务，避免本地或 CI 中残留 4173 端口进程。
首次运行 E2E 前如提示缺少浏览器，请执行 `npx playwright install chromium`。

## 数据与隐私

- 文档、示范文本、执笔人版本、测试记录和接口配置默认保存在本机浏览器 IndexedDB。
- 移入垃圾箱的文档仍保存在本机浏览器中，并通过右上角垃圾箱窗口集中恢复或清除；只有“清除 / 清空垃圾箱”才会从工作台数据中移除。
- `localStorage` 只保存主题、启动摘要和 IndexedDB 不可用时的兼容数据。
- 项目没有自带服务器，也不会把数据上传到项目作者处。
- 调用 AI 时，当前任务、相关文档片段、示范分析或执笔人规则会发送到你配置的 AI 接口。
- 处理敏感材料前，请确认接口提供方、组织制度和合规要求。

## 当前限制

- `.doc` / `.ppt` 旧版二进制 Office 文件暂不支持直接读取。
- PPT 生成支持电子杂志、瑞士国际主义、公文汇报、校园培训和自定义风格；自定义风格可保存为 PPT 执笔人，HTML 预览可在放大弹层中查看。
- 浏览器直接保存到真实文件夹依赖 File System Access API，不同浏览器支持程度不同。
- 当前没有内置后端代理，API Key 保存在使用者自己的浏览器环境中。
