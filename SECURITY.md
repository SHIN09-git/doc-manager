# 安全说明

摹文拟笔工作台是本地优先的静态前端应用。项目本身不内置 API Key，不提供默认云端存储，也不会把用户文档上传到项目作者处。

## 数据存储

默认存储位置：

- 文档库：浏览器 IndexedDB
- 执笔人、示范文本、版本和测试记录：浏览器 IndexedDB
- 真实文件夹句柄：浏览器授权存储
- 主题和启动摘要：`localStorage`
- IndexedDB 不可用时的兼容数据：`localStorage`

你可以通过右上角“备份”导出工作台数据。导出的备份文件可能包含文档正文、示范文本、执笔人规则和测试记录，请妥善保存。

## API Key

- 项目不附带任何 API Key。
- 请不要把真实 API Key 提交到仓库、Issue、截图或演示视频中。
- API Key 只应在你自己的浏览器环境中配置。
- 共享演示环境时，建议使用临时 Key、低权限 Key 或单独项目 Key。

## AI 调用数据

调用 AI 时，以下内容可能会发送到你配置的接口：

- 你在提示词窗口输入的内容
- 当前文档片段或全文
- 被 `@` 调用的执笔人规则
- 执笔人示范文本的分析结果
- PPT 生成所用的输入材料和结构要求

处理以下材料前，请先确认接口提供方、组织制度和合规要求：

- 个人身份信息
- 学生、员工、客户或患者资料
- 合同、财务、采购和薪酬信息
- 内部会议纪要、未公开政策或商业计划
- 任何受法律、制度或保密协议约束的内容

## 文件导入

项目会在浏览器本地读取用户选择或拖入的文件。当前支持：

- `.txt`
- `.md`
- `.text`
- `.csv`
- `.docx`
- `.pptx`

旧版 `.doc` / `.ppt` 暂不直接解析。请先用 Word、PowerPoint 或 WPS 另存为 `.docx` / `.pptx`。

## 真实文件夹授权

真实文件夹关联依赖浏览器 File System Access API。

- 授权由浏览器控制。
- 项目只在用户主动关联或同步时读取文件。
- 浏览器可能在重新打开页面后要求再次确认权限。
- 保存当前文档到真实文件夹时，会写入同名 `.txt` 文件。

## 依赖安全

当前主要依赖：

- `mammoth`：读取 `.docx`
- `jszip`：读取和检查 Office Open XML 包
- `pptxgenjs`：生成原生 `.pptx`
- `esbuild`：开发构建

提交依赖更新时，请说明用途，并运行：

```bash
npm audit
npm test
npm run build
```

## 漏洞报告

请通过 Issue 报告安全问题，并提供：

- 问题类型
- 复现步骤
- 影响范围
- 建议修复方向

不要公开真实 API Key、私密文档、可识别个人信息或组织内部敏感材料。
