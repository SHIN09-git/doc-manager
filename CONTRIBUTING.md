# 贡献指南

感谢你愿意参与摹文拟笔工作台。这个项目的目标是做一个本地优先、可直接使用、适合中文正式文本写作的开源工作台。

## 开发流程

1. 安装依赖：

```bash
npm install
```

2. 修改代码后重新构建：

```bash
npm run build
```

3. 提交前检查：

```bash
npm run check
npm test
```

## 代码约定

- 优先保持静态前端可直接打开，不引入不必要的后端依赖。
- 用户可见文案使用“执笔人”，内部函数名可以继续使用既有英文命名，避免无意义的大规模重命名。
- AI 链路要保留 JSON 容错、失败重试和用户友好的错误提示。
- 执笔人构建必须坚持“单篇样本只产生候选规则，多篇共同验证后才成为强规则”。
- 不要把样本文档中的具体人名、时间、地点、活动名称、临时安排或隐私信息提升为通用规则。

## 提交建议

- 一个 Pull Request 尽量只解决一类问题。
- UI 修改请说明主要交互变化，并尽量附上截图。
- AI 提示词修改请说明对输出结构、隐私过滤或规则置信度的影响。
- 涉及存储结构时，请说明是否需要迁移旧数据。

## 问题反馈

提交 Issue 时请尽量提供：

- 运行方式：直接打开 `index.html` 或本地服务
- 浏览器与系统版本
- 复现步骤
- 预期结果与实际结果
- 控制台错误截图或日志
