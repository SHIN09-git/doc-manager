# 快速开始

这份文档面向第一次打开项目的人。如果你只想使用工作台，不需要理解源码。

## 运行方式

### 直接打开

双击项目根目录里的 `index.html`。

适合：只想试用、演示或在本机管理少量文档。

### 本地服务器

如果浏览器限制 `file://` 页面，可以启动一个静态服务器。

Windows：

```bat
start.bat
```

macOS / Linux：

```bash
./start.sh
```

默认访问：

```text
http://localhost:8000
```

## 首次配置 AI

1. 打开页面右上角“接口”。
2. 填写 OpenAI 兼容接口信息。
3. 点击“保存接口”。
4. 点击“测试”，确认接口能返回内容。

常见字段：

- `Base URL`：接口基础地址。
- `Endpoint Path`：通常是 `/chat/completions`。
- `Model`：模型名称。
- `API Key`：你的接口密钥，只保存在本机浏览器。
- `System Prompt`：默认写作角色设定。

## 管理文档

你可以通过三种方式加入文档：

1. 点击右上角“导入”。
2. 把文件拖入左侧文档区。
3. 关联真实本地文件夹后导入其中的文件。

支持格式：

- `.txt`
- `.md`
- `.text`
- `.csv`
- `.docx`
- `.pptx`

`.docx` 和 `.pptx` 中的表格会被转换为 Markdown 表格，便于 AI 理解。

左侧文档卡片也可以直接拖到右侧“生成”提示词窗口，系统会把该文档标题和正文追加为本次起草素材。

## 编辑文档

中间区域是当前文档编辑器：

- 标题在正文上方。
- 正文可直接人工编辑。
- “保存当前文档”会保存到本机浏览器数据区。
- “查找替换”可以展开或收起。
- 在正文中右键可以复制、删除、段落 AI 重写或整理格式。
- 拖动编辑区两侧的分隔条，可以调整中间编辑区域宽度。

导出当前文档默认生成 `.docx` Word 文件；如果正文第一行和文档标题重复，例如 AI 生成的 `**大标题**`，导出时会自动去掉正文里的重复标题。

## 构建执笔人

执笔人用于稳定复用某一类文本的结构、文风、格式和审稿规则。

推荐流程：

1. 打开右侧“执笔人”。
2. 新建一个执笔人，例如“通知写作”或“工作总结”。
3. 添加 3-8 篇同类示范文本，建议至少 5 篇。
4. 点击构建，让 AI 完成单篇分析、多篇聚合、草案生成和测试。
5. 查看质量报告、版本和测试结果。
6. 人工修改规则或添加反馈后继续优化。

使用时，在 AI 起草窗口或正文编辑器中输入：

```text
@通知写作 请起草一份关于培训安排的通知
```

## 生成 PPTX

打开右侧“PPT”栏目：

1. 输入标题。
2. 选择风格并填写页数；默认包含归藏杂志、瑞士网格、公文汇报、校园培训等风格，也可以勾选“自动”让 AI 根据素材信息量决定页数。
3. 粘贴大纲、文档内容，或拖入 `.pptx` / `.docx` / 文本文件作为素材。
4. 点击“生成 PPTX 草稿”。
5. 检查或修改生成的幻灯片 JSON 结构和结构自检报告。
6. 可用“放大预览”查看更大的 HTML 结构预览，再点击“下载 PPTX”导出可编辑的 PowerPoint 文件。

当前 PPT 生成不是网页截图，而是原生 `.pptx`：标题、正文、列表、表格和备注都可以在 PowerPoint / WPS 中继续编辑。

## 备份与隐私

- 点击右上角“备份”可以导出完整工作台数据。
- 文档和执笔人默认保存在本机浏览器 IndexedDB。
- API Key 只保存在当前浏览器环境。
- 调用 AI 时，相关文本会发送到你配置的接口，请先确认敏感材料处理要求。

## 开发者命令

如果你修改了源码：

```bash
npm install
npm run build
npm run check
npm test
npm run test:e2e
```

浏览器加载的是 `build/bundle.js`。修改 `src/` 之后，一定要重新构建。
首次运行 E2E 前如提示缺少浏览器，请执行 `npx playwright install chromium`。
