# 上手指南

这份文档面向第一次使用“摹文拟笔工作台”的用户。你可以把它当作从 0 到生成第一篇文档、训练第一个执笔人的操作流程。

## 1. 打开工作台

推荐使用本地服务：

```bash
npm install
npm run build
node e2e/serve-static.mjs
```

浏览器访问：

```text
http://127.0.0.1:4173/index.html
```

如果只是快速试用，也可以直接双击 `index.html`。部分浏览器会限制本地文件能力，本地服务方式更稳定。

## 2. 配置 AI 接口

点击右上角“接口”，填写你的 OpenAI 兼容接口配置：

- Base URL
- Endpoint Path，通常为 `/chat/completions`
- Model
- API Key
- 可选系统提示词

配置只保存在当前浏览器。项目没有内置后端，也不会替你保存 API Key。

## 3. 导入和管理文档

左侧是文档管理栏。

你可以：

- 点击或拖入文档到导入区域。
- 关联真实文件夹，并在工作台中维护自定义标签。
- 点击文档后在中间编辑区查看内容。
- 拖动文档调整顺序，或在菜单中置顶、置底、删除。
- 从右上角垃圾箱恢复误删除文档，或彻底清除。

支持稳定阅读的格式：

- `.txt`
- `.md`
- `.text`
- `.csv`
- `.docx`
- `.pptx`

老式 `.doc` 和 `.ppt` 建议先另存为 `.docx` 和 `.pptx`。

## 4. 使用执笔人

右侧默认打开“执笔人”窗口。

执笔人是工作台的核心能力。它不是简单的模板，而是从多篇同类文档中提炼出的写作规则，包括：

- 文档结构
- 标题和开头方式
- 段落功能
- 常用句式
- 文风语气
- 禁止复用的信息
- 审稿自检标准

点击执笔人卡片可以展开详情。未选中时，卡片只保留名称和“调用”按钮，方便快速选择。

点击“调用”后：

1. 工作台会切到“生成”窗口。
2. 提示词框里会自动插入对应的 `@调用名`。
3. 你可以继续补充本次要写的具体内容。

## 5. 生成执笔人

点击“生成执笔人”打开二级窗口。

### 基础信息

填写：

- 名称
- 分类
- 启用状态

如果分类选择“自定义”，自定义输入框会自动展开。

一句话能力描述不需要手工填写，系统会在生成后根据样本自动提炼。

### 添加训练样本

训练样本可以来自两个位置：

- 已加入左侧文档库的文件
- 本地上传的新文件

建议上传 3 到 5 篇同类正式文档。单篇文档只能产生候选规则，多篇文档共同验证后才会形成强规则。

### 构建流程

点击“开始生成”后，弹窗会关闭，执笔人卡片会进入构建状态，并显示步骤进度：

1. 单篇文档解析
2. 多篇规则聚合
3. 生成执笔人草案
4. 自动测试与规则命中检查
5. 保存版本和质量报告

如果失败，卡片会显示失败原因，并提供重试入口。

## 6. 起草文档

切到“生成”窗口后，可以选择：

- 生成新文档
- 插入到当前文档
- 覆盖当前文档

提示词中可以使用 `@调用名` 指定执笔人。例如：

```text
@学校通知 请写一份关于下周教研活动安排的通知，语气正式，控制在 800 字以内。
```

生成过程中可以取消任务。遇到限流、超时或 JSON 解析异常时，系统会尝试重试并给出提示。

## 7. 编辑和导出

中间是文本编辑区。

常用操作：

- 手动编辑正文和标题
- 拖动边界调整编辑区宽度
- 撤销编辑
- 查找和替换
- 右键选中文本进行复制、删除、段落重写、格式整理
- 使用右键菜单选择执笔人进行段落改写

导出当前文档时，默认导出为 Word 文档。导出时会避免把正文里加粗的大标题重复写进文件。

## 8. 生成 PPT

进入 PPT 生成窗口，可以把文本、Word、PPTX 或左侧文档拖入素材区。

支持：

- 自定义页数
- 根据内容自动调整页数
- 生成原生可编辑 `.pptx`
- 预览页面放大查看
- 公文汇报、校园培训、归藏风格、瑞士风和自定义风格

## 9. 备份和迁移

建议定期使用备份功能导出工作台数据。执笔人也支持单独导入和导出，便于在团队之间共享。

导出的执笔人包会包含规则、版本、说明和可选样本摘要。请确认其中不包含不应共享的敏感内容后再发送给他人。

## 10. 常见问题

### 导入后正文乱码

工作台会自动尝试多种中文编码。如果仍然乱码，建议先用文本编辑器另存为 UTF-8，再导入。

### Word 无法导入

当前稳定支持 `.docx`。如果是 `.doc`，请在 Word 或 WPS 中另存为 `.docx`。

### PPT 拖入没有内容

请确认文件是 `.pptx`。老式 `.ppt` 需要先转换。

### AI 没有响应

先检查接口配置是否正确，再查看是否遇到 401、429、超时或网络错误。工作台会给出对应提示。

### 换浏览器后数据不见了

数据默认保存在当前浏览器。换浏览器或换设备前，请先导出备份。
