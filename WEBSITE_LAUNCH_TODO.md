# 网站上线未完成工作与 TODO

更新日期：2026-06-09

本文档面向一个明确目标：把“摹文拟笔工作台”从本地可用产品推进到真实网站。上线后的最低形态是：用户可以通过域名访问、注册登录、使用 AI、查看套餐与额度、提交人工充值申请；管理员可以进入后台审核充值、查看用量、处理反馈、排查错误和维护数据。

## 一、当前结论

项目已经具备灰度试运营的产品和工程底座，但还没有完成真实网站上线所需的外部资源接入、生产环境联调和运营规则确认。

已经完成的基础能力：

- 用户工作台：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、云端账户、套餐额度、费用明细、人工充值申请。
- 管理后台：用户与组织管理、成员权限、组织接口、用量统计、成本估算、审计日志、反馈处理、错误跟进、人工充值审核。
- 后端能力：注册登录、会话、额度、套餐、人工充值、AI 代理、邮件适配、审计、备份、PostgreSQL Store、Docker、部署检查、GitHub Actions。
- 安全与稳定性：生产环境变量检查、敏感字段脱敏、备份校验、Playwright 端到端测试、单元测试和构建检查。

当前最适合的上线方式：

- 先做小范围灰度，不急着接自动支付。
- 采用“VPS + Docker Compose + Nginx + Cloudflare + PostgreSQL + 人工确认充值”。
- 首批邀请 3 到 20 个真实用户试用，观察稳定性、AI 成本、充值转化和真实反馈。

### P1：灰度上线后尽快完成

这些事项影响运营效率和用户体验，可以在灰度期间继续补齐。

- 新用户引导：注册、登录、云端同步、充值、额度不足、执笔人训练和 PPT 生成需要形成连续路径。
- 站内通知：充值提交、审核通过、审核拒绝、额度不足、邮件失败、AI 失败需要有更明确的用户提示。
- 帮助文档：快速上手、充值说明、执笔人训练说明、PPT 生成说明、常见问题。
- 管理员手册：充值审核、反馈处理、错误跟进、用户处理、备份恢复、套餐调整。
- 运营日报：每日注册、活跃、用量、充值、成本、错误、反馈摘要。
- 监控告警：API 5xx、AI 调用失败、邮件失败、备份失败、数据库容量、额度异常。
- 默认云端地址：部署后前端默认使用正式 `/api`，减少用户手动配置。
- 数据导出与留档：用户数据导出、组织数据导出、审计 CSV、费用明细 CSV 需要在真实环境验收。

### P2：公开收费版本需要完成

这些事项属于更正式的商业化，不建议压在第一次灰度上线前。

- 自动支付：接入微信支付、支付宝、Stripe、Paddle、Creem 等至少一种渠道。
- 支付对账：订单号、渠道回调、重复回调幂等、退款、内部额度流水一致性。
- 发票与财务：发票规则、收入记录、账单导出、税务处理流程。
- 客服工单：问题编号、负责人、状态、SLA、消息通知和跟踪记录。
- 团队套餐：组织共享额度、成员权限、团队执笔人共享、团队文档空间。
- 数据治理：账号删除、组织删除、数据保留周期、训练样本脱敏、敏感材料提醒。
- 安全运营：生产密钥轮换、漏洞反馈渠道、安全响应流程、访问日志留存策略。
- 公开营销页：产品介绍、价格页、案例页、帮助中心、服务条款和隐私政策入口。
