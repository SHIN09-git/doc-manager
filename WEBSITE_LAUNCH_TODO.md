# 网站上线未完成工作与总控 TODO

更新时间：2026-06-09

本文档面向“把摹文拟笔工作台部署成真实可访问、可注册、可充值、可运营的网站”这一目标，整理当前未完成事项、上线优先级，以及项目负责人需要准备的资源。

## 当前状态

项目已经具备小规模灰度试运营的产品和工程底座，但还没有完成公开收费网站上线所需的生产资源接入和实地演练。

已具备：

- 本地工作台：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、导入导出、备份和垃圾箱。
- 云端用户体系：注册、登录、邮箱验证、密码重置、组织、成员、角色和云端同步。
- 商业化底座：AI 代理、额度统计、人工确认充值、费用明细、审计、反馈、错误跟进和独立管理员后台。
- 部署基础：静态前端构建、Node API、PostgreSQL Store、迁移脚本、Dockerfile、Docker Compose、Render 蓝图草案、生产环境变量校验。
- 运维基础：健康检查、生产配置检查、逻辑备份、加密备份、备份验证和备份恢复演练脚本。

仍未完成：

- 生产域名、服务器、数据库、邮件、AI Key、收款码等真实资源尚未接入。
- PostgreSQL 真实生产库还需要完整跑一遍注册、登录、同步、充值、后台确认和备份恢复演练。
- 人工充值链路已有产品功能，但需要接入真实微信/支付宝收款码和正式套餐规则。
- 邮箱验证和密码重置需要接真实邮件服务并完成发信域名认证。
- 隐私政策、用户协议、退款规则、发票规则和使用边界需要由运营主体最终确认。
- 公开收费版本还缺自动支付、对账、发票、客服工单和更完整监控告警。

### P1：灰度上线后尽快完成

目标：降低用户使用门槛，提升灰度运营效率。

- 默认云端 API 地址产品化：正式网站自动连接 `https://api.<domain>/api`，减少用户手填。
- 新用户引导：注册、邮箱验证、云端同步、套餐充值、额度不足提示形成完整路径。
- 站内通知：充值提交、审核通过、审核拒绝、额度不足、邮箱状态变化能主动提示用户。
- 后台运营增强：反馈和错误批量导出、运营日报、常用筛选、SLA 风险看板继续打磨。
- 监控告警：API 5xx、AI 代理失败、邮件失败、备份失败、数据库容量和连接数告警。
- 用户文档：用户手册、管理员手册、充值说明、数据备份说明和常见问题。

### P2：公开收费版本需要完成

目标：从人工灰度运营升级到可规模化收费运营。

- 自动支付：微信支付、支付宝、Stripe、Paddle、Creem 等至少接入一种正式渠道。
- 支付回调和对账：签名校验、订单状态、退款、关闭订单、重复回调幂等处理。
- 发票与财务：发票规则、账单导出、渠道订单和内部额度流水对账。
- 更细团队套餐：团队共享执笔人、组织文档空间容量、成员权限细分、团队账单信息。
- 数据治理：数据保留周期、组织级删除流程、管理员二次确认、敏感样本脱敏提示。
- 客服体系：客服入口、工单编号、处理 SLA、常见问题和服务状态页。
- 法务与安全：隐私合规复核、安全响应流程、漏洞反馈渠道和生产密钥轮换制度。
