# 网站上线未完成工作与 TODO

更新日期：2026-06-09

本文档用于把“摹文拟笔工作台”从本地可运行项目推进到真实网站。当前目标不是一次做成完整大型 SaaS，而是先上线一个可控灰度版本：用户能访问域名、注册登录、使用 AI、查看套餐和费用明细、提交人工充值；管理员能确认充值、处理反馈、排查错误、查看用量成本并维护数据。

## 1. 当前结论

项目已经具备灰度试运营底座，但还不建议直接大规模公开收费。

已经完成的关键能力：

- 用户工作台：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、本地备份、云端账户、套餐额度、费用明细、人工充值申请。
- 管理后台：独立 `admin.html`，支持组织、成员、接口、用量、成本估算、审计、反馈、错误跟进、邮件记录和人工充值审核。
- 后端底座：注册登录、会话、邮箱验证、密码重置、额度、套餐、人工充值、AI 代理、邮件适配、审计、备份、PostgreSQL Store、Docker 和部署检查。
- 工程质量：已有构建检查、单元测试、端到端测试、GitHub Actions、生产环境变量校验、敏感字段脱敏和备份校验。

仍未完成的关键闭环：

- 还没有接入真实域名、真实服务器、生产 PostgreSQL、正式 AI Key、邮件服务和 HTTPS 收款码。
- 还没有在真实生产环境完整验收注册、登录、邮件、AI、充值、后台确认、额度到账、费用明细、审计、备份和恢复。
- 套餐价格、额度扣减、退款规则、客服入口、隐私政策、用户协议和 AI 免责声明需要最终确认。
- 自动支付、发票、企业 SLA、复杂团队协作、公开营销站和客服工单系统可以后置。

推荐第一版上线形态：

- 部署：`VPS + Docker Compose + Nginx + Cloudflare + PostgreSQL`。
- 充值：先使用微信/支付宝收款码，管理员后台人工确认到账。
- 灰度：邀请 3 到 20 个真实用户试用 1 到 2 周。
