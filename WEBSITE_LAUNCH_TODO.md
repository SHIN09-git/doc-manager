# 网站上线剩余工作与 TODO

更新日期：2026-06-09

本文档用于统筹“摹文拟笔工作台”从本地可用产品走向真实网站灰度上线的剩余工作。目标不是立刻做成完整 SaaS，而是先上线一个可控灰度版：用户能通过域名访问、注册登录、使用 AI、提交人工充值、查看额度和费用明细；管理员能确认充值、处理反馈、排查错误、查看用量成本并维护数据安全。

## 1. 当前结论

当前版本已经具备灰度试运营底座，但还不建议直接大规模公开收费。

已经完成：

- 工作台核心功能：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、本地备份、云端账户、套餐额度、费用明细、人工充值申请。
- 管理后台：独立 `admin.html`，支持组织、成员、接口、用量、成本估算、审计、反馈、错误跟进、邮件记录和人工充值审核。
- 后端基础：注册登录、会话、邮箱验证、密码重置、组织、额度、套餐、人工充值、AI 代理、邮件适配、审计、备份、PostgreSQL Store、Docker 和部署检查。
- 工程质量：已有构建检查、单元测试、端到端测试、GitHub Actions、生产环境变量校验、敏感字段脱敏、备份校验和恢复演练脚本。

仍未完成：

- 真实生产入口未落地：正式域名、DNS、HTTPS、主站地址和 API 地址还没有接入。
- 真实生产环境未跑通：VPS 或托管平台、生产 PostgreSQL、Nginx/反向代理、生产环境变量还没有在真实机器上验收。
- 第三方服务未真实联调：AI Key、邮件服务、收款码 HTTPS 图片地址、邮件投递回调、备份告警都需要用真实账号验证。
- 商业规则未最终确认：套餐价格、额度扣减、充值到账、退款规则、客服入口、运营主体、隐私政策、用户协议、AI 免责声明需要最终确认。
- 运营闭环未真实演练：灰度名单、管理员分工、充值审核流程、每日巡检、备份恢复演练还没有在真实环境执行。

推荐第一版上线形态：

- 部署：`VPS + Docker Compose + Nginx + Cloudflare + PostgreSQL`。
- 充值：先使用微信/支付宝收款码，管理员后台人工确认到账。
- 灰度：邀请 3 到 20 个真实用户试用 1 到 2 周。
- 暂缓：自动支付、发票系统、复杂客服工单、公开营销官网、执笔人市场、复杂团队协作。

## 3. 你需要优先准备的事项

### 3.1 域名与访问入口

- [ ] 确定正式域名。
- [ ] 提供 DNS 管理权限，推荐接入 Cloudflare。
- [ ] 确认是否主要面向中国大陆用户；如果使用大陆服务器，需要准备 ICP 备案。
- [ ] 确认主站域名和 API 域名方案，例如 `mowen.example.com` 与 `api.mowen.example.com`。

### 3.2 服务器与数据库

- [ ] 购买或提供一台 VPS/云服务器，第一版建议 2 核 4G 起步。
- [ ] 提供服务器登录方式，建议使用 SSH 密钥。
- [ ] 决定 PostgreSQL 方案：VPS Docker 内置，或云数据库。
- [ ] 确认备份保存位置：服务器本地、对象存储，或两者都要。
- [ ] 确认备份保留周期，例如 14 天或 30 天。

### 3.3 AI、邮件、收款

- [ ] 提供 AI 服务商、API Key、模型名称、预算上限。
- [ ] 确认 AI 额度扣减规则：一次文本生成、一次执笔人训练、一次 PPT 生成各扣多少。
- [ ] 注册邮件服务账号，推荐 Resend，并完成发信域名验证。
- [ ] 提供微信和支付宝收款码图片，最好是可公开访问的 HTTPS 图片地址。
- [ ] 确认人工充值时用户需要填写的付款备注规则。

### 3.4 套餐与商业规则

- [ ] 确认免费版每日额度。
- [ ] 确认 Pro 套餐价格、额度、有效期。
- [ ] 确认 Team 套餐价格、额度、有效期。
- [ ] 确认单独额度包价格和额度。
- [ ] 确认人工充值最低金额。
- [ ] 确认退款规则：什么情况下退、退到哪里、已消耗额度如何处理。
- [ ] 确认是否需要开发票；灰度期建议先人工处理。

### 3.5 运营与合规

- [ ] 确认运营主体名称。
- [ ] 提供客服邮箱、微信、QQ 群或飞书群。
- [ ] 指定至少一名管理员负责充值审核。
- [ ] 指定至少一名管理员负责错误和反馈跟进。
- [ ] 最终确认隐私政策、用户协议、AI 生成内容免责声明。
- [ ] 准备第一批灰度用户名单和典型使用场景。

- [ ] 根据真实域名整理生产 `.env`。
- [ ] 配置 Docker Compose、Nginx、Cloudflare DNS 和 HTTPS。
- [ ] 部署前端静态站和后端 API。
- [ ] 接入 PostgreSQL，执行迁移并验证数据持久化。
- [ ] 接入正式 AI 代理，验证用量、成本、额度扣减和失败记录。
- [ ] 接入 Resend 邮件发送与回调，验证邮箱验证和密码重置。
- [ ] 配置人工充值套餐、收款码、后台审核、额度到账、费用明细和审计流水。
- [ ] 配置备份、加密备份、备份校验、对象存储上传和恢复演练。
- [ ] 跑完整业务验收：注册、登录、文档、执笔人、AI、PPT、充值、后台、备份。

我不能替你完成：

- [ ] 购买域名或服务器。
- [ ] 完成 ICP 备案、企业认证、商户号认证。
- [ ] 长期保管生产密钥。
- [ ] 承担法律、财税、隐私合规责任。
- [ ] 处理真实收款、退款、发票和客户纠纷。

- [ ] 你提供服务器或托管平台账号。

## 8. 暂时建议推迟

- 自动支付渠道接入：先用人工确认充值灰度，确认真实付款意愿后再接微信支付、支付宝开放平台或国际支付渠道。
- 大规模公开营销页：先保证真实用户能稳定完成核心流程，再扩展官网、案例页和价格页。
- 复杂团队协作和发票财务系统：灰度期先用管理员后台和人工流程兜底。
- 执笔人市场或社区共享：等核心生成、计费、数据安全和客服流程稳定后再做。
