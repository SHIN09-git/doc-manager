## 1. 当前结论

项目已经具备小范围灰度试运营的产品和工程底座，但还不建议直接公开收费上线。

已经完成：

- 用户工作台：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、云端账户、套餐额度、费用明细、人工充值申请。
- 管理后台：用户与组织管理、成员权限、组织接口、用量统计、成本估算、审计日志、反馈处理、错误跟进、人工充值审核。
- 后端底座：注册登录、会话、额度、套餐、人工充值、AI 代理、邮件适配、审计、备份、PostgreSQL Store、Docker、部署检查、GitHub Actions。
- 安全与稳定性：生产环境变量检查、敏感字段脱敏、备份校验、Playwright 端到端测试、单元测试和构建检查。

仍未完成的关键点：

- 没有接入真实域名、VPS、PostgreSQL、邮件服务、AI Key 和收款码。
- 没有在真实生产环境跑完整注册、登录、邮件、AI、充值、后台确认、额度到账和备份恢复验收。
- 套餐价格、额度规则、退款规则、客服入口、隐私政策和用户协议仍需要最终确认。
- 当前适合先做“人工确认充值 + 小范围灰度”，不建议一开始就做自动支付和大规模开放。

推荐第一版上线方式：

- VPS + Docker Compose + Nginx + Cloudflare + PostgreSQL。
- 充值先用微信/支付宝收款码，管理员后台人工确认到账。
- 首批邀请 3 到 20 个真实用户试用 1 到 2 周。

### P1：灰度上线后尽快完成

- 新用户引导：注册、登录、云端同步、充值、额度不足、执笔人训练、PPT 生成形成连续路径。
- 站内通知：充值提交、审核通过、审核拒绝、额度不足、邮件失败、AI 失败需要更明确的用户提示。
- 帮助文档：快速上手、充值说明、执笔人训练说明、PPT 生成说明、常见问题。
- 管理员手册：充值审核、反馈处理、错误跟进、用户处理、备份恢复、套餐调整。
- 运营日报：每日注册、活跃、用量、充值、成本、错误、反馈摘要。

- 默认云端地址：部署后前端默认使用正式 `/api`，减少用户手动配置。
- 数据导出与留档：用户数据导出、组织数据导出、审计 CSV、费用明细 CSV 在真实环境验收。
