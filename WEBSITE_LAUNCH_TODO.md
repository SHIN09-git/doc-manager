# 真实网站上线剩余工作与 TODO

更新日期：2026-06-09

本文用于整理“摹文拟笔工作台”从本地项目走向真实网站时仍未完成的工作，并明确哪些事项需要项目负责人准备，哪些事项可以继续由 Codex 实施。

当前建议目标不是一次性做成完整 SaaS，而是先上线一个可控灰度版本：用户能通过域名访问、注册登录、使用 AI、提交人工充值、查看额度和费用明细；管理员能确认充值、查看用量成本、处理反馈和错误，并保证数据可备份、可恢复、可审计。

## 1. 当前完成情况

项目已经具备灰度试运营的产品和工程底座，但还不建议直接大规模公开收费。

已经具备：

- 本地工作台：文档管理、文本编辑、AI 起草、执笔人构建与调用、PPT 生成、导入导出、备份。
- 云端账号：注册登录、邮箱验证、密码重置、组织、成员、文档同步、执笔人同步、AI 代理、用量统计、审计记录。
- 商业化底座：套餐、额度、费用明细、人工确认充值、支付 webhook 预留、后台管理、反馈和错误跟进。
- 运维底座：PostgreSQL Store、Docker Compose、生产环境变量校验、备份脚本、备份校验、CI、端到端测试。
- 管理后台：独立 `admin.html`，支持成员、接口、用量、审计、反馈、邮箱、账单、错误和运营跟进。
- 数据治理：账号删除、组织所有者交接、空组织资源归档、审计记录和隐私政策说明已经补齐基础逻辑。

仍未完全落地：

- 正式域名、DNS、HTTPS、服务器、Nginx/Cloudflare 访问链路。
- 真实生产 PostgreSQL、迁移、备份、恢复演练。
- 真实 AI Key、模型、成本规则和额度扣减验收。
- 真实邮箱服务商域名认证、发信和回调联调。
- 微信/支付宝收款码、套餐价格、退款规则和人工审核流程。
- 运营主体、客服入口、隐私政策、用户协议、AI 内容免责声明的最终确认。
- 灰度用户名单、管理员分工和每日运营检查流程。

推荐第一版上线形态：

- 部署路径：VPS + Docker Compose + Nginx + Cloudflare + PostgreSQL。
- 充值路径：先使用微信/支付宝收款码 + 管理员后台人工确认充值。
- 试运营范围：先邀请 3 到 20 个真实用户灰度使用 1 到 2 周。
- 暂缓事项：自动支付、发票系统、公开营销官网、复杂工单系统、执笔人市场、大规模团队协作。

## 2. 你现在最先要做的 8 件事

如果要尽快把网站跑到真实域名上，建议你先按这个顺序准备资源：

- [ ] 确认域名，并把 DNS 接入 Cloudflare 或提供 DNS 管理权限。
- [ ] 购买或确定 VPS/云服务器，并提供 SSH 登录方式。
- [ ] 决定 PostgreSQL 使用 Docker 内置还是云数据库。
- [ ] 准备正式 AI API Key、模型名和每月预算上限。
- [ ] 准备邮箱服务账号，例如 Resend，并完成发信域名验证。
- [ ] 准备微信/支付宝收款码，以及套餐价格、额度、有效期、退款规则。
- [ ] 指定至少一名管理员负责充值审核、反馈处理和错误跟进。
- [ ] 确认运营主体、客服入口、第一批灰度用户名单和典型使用场景。

## 4. 你需要优先准备的资源

### 4.1 域名与访问入口

- [ ] 一个正式域名。
- [ ] DNS 管理权限，推荐接入 Cloudflare。
- [ ] 是否主要面向中国大陆用户；如果使用大陆服务器，需要按最新要求准备 ICP 备案。
- [ ] 主站域名和 API 域名方案，例如 `mowen.example.com` 与 `api.mowen.example.com`。

### 4.2 服务器与数据库

- [ ] 一台 VPS/云服务器，第一版建议 2 核 4G 起步。
- [ ] 服务器登录方式，建议使用 SSH 密钥。
- [ ] PostgreSQL 方案：VPS Docker 内置，或独立云数据库。
- [ ] 备份保存位置：服务器本地、对象存储，或两者都要。
- [ ] 备份保留周期，例如 14 天或 30 天。

### 4.3 AI、邮箱、收款

- [ ] AI 服务商、API Key、模型名称、预算上限。
- [ ] AI 额度扣减规则：文档生成、执笔人训练、PPT 生成分别扣多少额度。
- [ ] 邮箱服务账号，推荐 Resend 或同类服务，并完成发信域名验证。
- [ ] 微信和支付宝收款码图片，最好提供可公开访问的 HTTPS 图片地址。
- [ ] 人工充值时用户需要填写的付款备注规则。

### 4.4 套餐与商业规则

- [ ] 免费版每日额度。
- [ ] Pro 套餐价格、额度、有效期。
- [ ] Team 套餐价格、额度、有效期。
- [ ] 单独额度包价格和额度。
- [ ] 人工充值最低金额。
- [ ] 退款规则：什么情况下退、退到哪里、已消耗额度如何处理。
- [ ] 是否需要开发票；灰度期建议先人工处理。

### 4.5 运营与合规

- [ ] 运营主体名称。
- [ ] 客服邮箱、微信、QQ 群或飞书群。
- [ ] 至少 1 名管理员负责充值审核。
- [ ] 至少 1 名管理员负责错误和反馈跟进。
- [ ] 最终确认隐私政策、用户协议、AI 生成内容免责声明。
- [ ] 第一批灰度用户名单和典型使用场景。

> 备案、税务、退款和隐私合规属于法律/财务事项，正式收费前建议按最新监管和服务商要求复核。本文只作为工程与上线准备清单。

## 5. Codex 可以继续完成的工作

不依赖真实资源时，可以先做：

- [ ] 新用户上手引导：注册、登录、云端同步、充值、执笔人训练、PPT 生成。
- [ ] 帮助中心入口：快速上手、充值说明、执笔人训练说明、PPT 生成说明、常见问题。
- [ ] 管理员操作手册：充值审核、反馈处理、错误跟进、成员处理、备份恢复、套餐调整。
- [ ] 生产默认 API 地址策略：部署后前端默认使用正式 `/api`，减少用户手动配置。
- [ ] 上线自检脚本增强：检查 AI、邮箱、套餐、备份、CORS、安全 Cookie。
- [ ] 灰度运营日报模板：注册、活跃、AI 用量、充值、成本、错误、反馈摘要。
- [ ] 继续做高损失问题 review，优先修复真实用户、充值、后台和数据安全链路。

拿到真实资源后，可以继续做：

- [ ] 根据真实域名整理生产 `.env`。
- [ ] 配置 Docker Compose、Nginx、Cloudflare DNS 和 HTTPS。
- [ ] 部署前端静态站和后端 API。
- [ ] 接入 PostgreSQL，执行迁移并验证数据持久化。
- [ ] 接入正式 AI 代理，验证用量、成本、额度扣减和失败记录。
- [ ] 接入 Resend 或同类邮件发送与回调，验证邮箱验证和密码重置。
- [ ] 配置人工充值套餐、收款码、后台审核、额度到账、费用明细和审计流水。
- [ ] 配置备份、加密备份、备份校验、对象存储上传和恢复演练。
- [ ] 跑完整业务验收：注册、登录、文档、执笔人、AI、PPT、充值、后台、备份。

Codex 不能替代你完成：

- [ ] 购买域名、服务器或第三方账号。
- [ ] ICP 备案、企业认证、商户号认证。
- [ ] 长期保管生产密钥。
- [ ] 承担法律、财税、隐私合规责任。
- [ ] 处理真实收款、退款、发票和客户纠纷。

## 9. 上线闸门

以下任一项未通过，都不建议邀请真实付费用户：

- [ ] 正式域名 HTTPS 可访问。
- [ ] 注册、邮箱验证、登录、退出、密码重置可用。
- [ ] 文档导入、编辑、导出、执笔人训练、AI 生成、PPT 生成可用。
- [ ] 人工充值提交、管理员确认、额度到账、费用明细、审计记录可用。
- [ ] 管理后台充值、反馈、错误、用量、成本、邮箱、导出链路可用。
- [ ] 备份、备份校验、恢复演练至少成功一次。

## 10. 暂时建议推迟

- 自动支付渠道接入：先用人工确认充值灰度，确认真实付费意愿后再接微信支付、支付宝开放平台或国际支付渠道。
- 大规模公开营销页：先保证真实用户能稳定完成核心流程，再扩展官网、案例页和价格页。
- 复杂团队协作和发票财务系统：灰度期先用管理员后台和人工流程兜底。
- 执笔人市场或社区共享：等核心生成、计费、数据安全和客服流程稳定后再做。
